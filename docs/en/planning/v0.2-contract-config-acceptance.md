# V0.2 Contract and Configuration Acceptance

This page is the English entry for the Eva-CLI V0.2 acceptance record. The
full-detail source is:
[V0.2 契约与配置稳定验收记录](../../zh-CN/planning/V0.2契约与配置稳定验收记录.md).

V0.2 stabilizes side-effect-free contracts for configuration, policy, and
observability. It does not claim CLI or runtime behavior.

Accepted V0.2 evidence:

- `eva-config` loads policy documents and topic route tables.
- `ProjectConfig` now aggregates agents, adapters, capabilities, policies, and
  routes.
- Route targets are checked against known Agents.
- `eva-policy` implements `PermissionSet`, `SandboxPolicy`, `PolicyLayer`, and
  `EffectivePolicy`.
- `eva-observability` implements `TraceFields`, audit contracts, and metrics
  contracts.
- Module README files for `eva-config`, `eva-policy`, and
  `eva-observability` describe APIs, boundaries, tests, and handoff rules.

V0.3 should build the minimum CLI development loop on top of these contracts:
`eva config validate`, `eva doctor`, `eva inspect`, stable diagnostics, and exit
codes.
