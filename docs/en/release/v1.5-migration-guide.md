# Eva-CLI V1.5 Migration Guide

Date: 2026-07-04

V1.5 is compatible with V1.4. It adds `eva-release` and the `eva release ...`
command group, but it does not remove or rename V1.0-V1.4 commands.

## From V1.4 To V1.5

1. Rebuild the workspace:

   ```powershell
   cargo build --release
   ```

2. Re-run the V1.4 lifecycle smoke command:

   ```powershell
   cargo run -- upgrade check --output json
   ```

3. Run the aggregate V1.5 release check:

   ```powershell
   cargo run -- release check --output json
   ```

4. Review the security findings:

   ```powershell
   cargo run -- release security --output json
   ```

5. Compare performance budgets:

   ```powershell
   cargo run -- release perf --output json
   ```

6. Print the machine-readable migration guide:

   ```powershell
   cargo run -- release migration --from-version 1.4.0 --to-version 1.5.0 --output json
   ```

## Breaking Changes

None in V1.5.

## Operational Notes

- `.eva/tasks` remains a local diagnostic format, not a durable public database.
- `restore plan` still reports `apply_allowed:false`.
- `upgrade check` still does not start Supervisor or Runtime processes.
- `hardware bind --apply` still validates only the logical plan.
- V1.5 release checks are safe to run in CI because they do not mutate project
  state or invoke external providers.
