# Eva-CLI V1.0 Known Limitations

V1.0 is a core CLI release. It stabilizes the executable path that already exists, documents it, and gates it in CI. It does not claim that every architecture document is implemented.

## Supported In V1.0 Core

- `eva --version` and `eva version --output json`.
- `eva doctor`, `eva config validate`, and `eva inspect`.
- `eva run --example basic` through `in_memory_v1.0`.
- Local task diagnostics through `eva task status`, `eva task logs`, and `eva task cancel`.
- Structured JSON envelopes, trace fields, structured errors, and stable exit-code mapping.
- In-memory EventBus, Scheduler, AgentRuntime, controlled Lua host contract, builtin `config.lint` capability, dead-letter replay diagnostics, and task logs for the basic example.

## Not Supported In V1.0 Core

- Background daemon or long-lived task scheduler.
- Durable task database, durable crash recovery, durable redrive queue, or durable replay cursor.
- Real Lua VM execution, arbitrary Lua evaluation, real hot-reload VM swap, or multi-generation traffic draining.
- External Adapter execution, stdio/http transports, MCP client/server calls, Skill execution, or hardware adapter calls.
- Discovery scans, memory service, knowledge service, backup, restore, migration package, release snapshot, supervisor lifecycle, or process-level upgrade.
- Installer packages, signed binaries, release channels, or automatic update flow.
- Compatibility guarantees for `.eva/tasks`; it is a local diagnostic format only.

## Operational Notes

- V1.0 commands are synchronous and single-process.
- The basic example is intentionally narrow; it is the regression baseline for later runtime work.
- Advanced architecture documents remain design targets for V1.1+ and should not be read as V1.0 runtime guarantees.
- CI proves the source checkout path, not packaged distribution artifacts.
