# Eva-CLI V1.5 GitHub 发版计划

日期：2026-07-04
发版标签：`v1.5.0`
版本管理归属：GitHub tag 与 GitHub Release

## 目标

把 Eva-CLI V1.5.0 发布为由 GitHub 管理的源码版本。当前仓库已经达到
V1.5 发布加固检查点，因此版本权威交给 GitHub：

- 不可变版本记录是 annotated Git tag `v1.5.0`；
- 对外发布页面是该 tag 对应的 GitHub Release；
- 发布证据由 GitHub Actions 在 tag 推送后生成；
- V1.5 仍然是源码发布，不是打包二进制发行版。

## 范围

包含：

- 由 tag 触发的 GitHub Release workflow；
- Windows、Linux、macOS 三个平台的 Rust workspace 验证；
- 发布前构建并校验官网与文档；
- 来自 `eva release check/security/perf/migration` 的 V1.5 发布门禁证据；
- GitHub 自动管理的源码压缩包。

不包含：

- 签名安装包或打包二进制 artifact；
- 破坏性 `restore apply`；
- 真实 Supervisor 进程 handoff；
- 真实 MCP server 启动；
- 硬件 raw I/O；
- 生产级 benchmark 认证。

## 发版前置条件

推送 `v1.5.0` 之前，`main` 分支上的发版 commit 必须包含：

1. `Cargo.toml` package/workspace 版本为 `1.5.0`。
2. V1.5 发布说明、迁移指南、兼容性策略和发布加固文档。
3. `.github/workflows/release.yml` GitHub Release workflow。
4. 本地 working tree 干净。
5. 本地检查通过：
   - `cargo fmt --check`
   - `cargo clippy --workspace --all-targets -- -D warnings`
   - `cargo test --workspace`
   - `./scripts/build-site-i18n.ps1`
   - `./scripts/validate-i18n.ps1`
   - `cargo run -- release check --output json`
   - `cargo run -- release security --output json`
   - `cargo run -- release perf --output json`
   - `cargo run -- release migration --output json`

## 执行计划

1. 提交并推送文档与官网更新。
2. 提交并推送本发版计划和 `.github/workflows/release.yml`。
3. 创建 annotated tag：

```powershell
git tag -a v1.5.0 -m "Eva-CLI V1.5.0"
```

4. 推送 tag：

```powershell
git push origin v1.5.0
```

5. 由 GitHub Actions 执行 `GitHub Release` workflow。
6. GitHub Actions 负责验证：
   - tag 格式与 Cargo 版本一致；
   - Rust format、clippy、tests 和 V1.5 smoke 命令；
   - 官网多语言构建与 i18n 校验；
   - release check/security/perf/migration 发布证据。
7. GitHub Actions 为 `v1.5.0` 创建或更新 GitHub Release。

## 发版后验证

推送 tag 后，先从 git 侧验证：

```powershell
git ls-remote --tags origin v1.5.0
```

然后在 GitHub Actions 中确认 `GitHub Release` workflow 完成。本机没有
`gh` 时，远端 tag 是本地可验证的版本归属证据；Release 页面由远端 workflow
创建。

## 回滚与修复

如果 release workflow 在创建 GitHub Release 前失败：

1. 在 `main` 修复失败原因。
2. 删除本地 tag：

```powershell
git tag -d v1.5.0
```

3. 仅当尚未公开发布 GitHub Release 时删除远端 tag：

```powershell
git push origin :refs/tags/v1.5.0
```

4. 从修复后的 commit 重新创建并推送 `v1.5.0`。

如果 GitHub Release 已经公开存在，优先创建 `v1.5.1` 之类的 patch release，
不要重写公开历史。

## 职责拆分

| 职责 | 归属 |
| --- | --- |
| 语义版本号 | `Cargo.toml` 与发布文档 |
| 版本身份 | Git tag `v1.5.0` |
| 对外发布页 | GitHub Release workflow |
| 发布证据 | GitHub Actions 日志与 `release-evidence-*` artifact |
| 源码压缩包 | GitHub 自动生成 |
| 安装包/签名 | 后续版本范围 |
