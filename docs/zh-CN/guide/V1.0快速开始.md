# Eva-CLI V1.0 快速开始

本文是 Eva-CLI V1.0 core 的受支持 quickstart。它验证一个干净 checkout 可以构建 CLI、校验配置、检查 runtime 边界、运行 basic Agent 闭环，并读取本地 task 诊断报告。

## 前置条件

- Git。
- 稳定 Rust toolchain，且 `cargo`、`rustfmt`、`clippy` 在 `PATH` 中可用。
- 只有本地运行网站脚本时才需要 PowerShell 7+。Rust quickstart 可在 PowerShell、Bash 或任意可执行 `cargo` 的 shell 中运行。

V1.0 core 不需要外部服务、后台 daemon、数据库、Lua VM 包、MCP server、硬件驱动或 API key。

## 从源码安装

```powershell
git clone https://github.com/Yetmos/Eva-CLI.git
cd Eva-CLI
cargo build --release
```

构建产物位于 Linux/macOS 的 `target/release/eva` 或 Windows 的 `target/release/eva.exe`。开发和验证时可以直接使用 `cargo run -- ...`，不必先把二进制加入 `PATH`。

## 验证发布面

```powershell
cargo run -- --version
cargo run -- version --output json
cargo run -- doctor --output json
cargo run -- config validate --output json
cargo run -- inspect runtime --output json
```

预期信号：

- `eva --version` 输出 `eva 1.0.0` 和 `release: V1.0 core`。
- `eva version --output json` 使用标准 success envelope，`runtime_mode` 为 `in_memory_v1.0`。
- `doctor` 报告配置根、schema、runtime boundary 和 Lua host boundary。
- `config validate` 加载 `config/eva.yaml`、manifests、policies、routes 和 schema 路径。
- `inspect runtime` 输出诊断用 no-op runtime 摘要。

## 运行 basic 闭环

```powershell
cargo run -- run --example basic --task-id req-quickstart-v10 --output json
```

V1.0 basic loop 是同步单进程路径：

1. CLI 加载 `examples/basic/config/eva.yaml`。
2. Runtime 构造 `in_memory_v1.0` core service table。
3. EventBus 接收 `/input/user` 并 append event。
4. Scheduler 把事件路由到 `root-agent`。
5. AgentRuntime 应用 timeout/cancel/retry 控制。
6. Lua host 校验受控 `on_event` table contract。
7. Capability router 调用 builtin `config.lint`。
8. CLI 写入 `.eva/tasks/req-quickstart-v10.task` 和 `.eva/tasks/latest-basic.task`。

## 查看 task 诊断

```powershell
cargo run -- task status --task req-quickstart-v10 --output json
cargo run -- task logs --task req-quickstart-v10 --output json
cargo run -- task cancel --task req-quickstart-v10 --reason quickstart-check --output json
```

已完成 task 上的 `task cancel` 只记录 cancel request 作为诊断证据，不会把终态 `completed` 改成 cancelled。

## 验证失败路径

```powershell
cargo run -- run --example basic --task-id req-quickstart-timeout --timeout-ms 0 --replay-dead-letters --output json
cargo run -- run --example basic --task-id req-quickstart-cancel --cancel --output json
```

timeout 命令应产生 `timed_out` task、一个 dead letter 和一个 replay summary。cancel 命令应产生 `cancelled` task、dead-letter 诊断，并且不会执行 Lua/capability。

## 发布门禁

修改 V1.0 core 行为前至少运行：

```powershell
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
./scripts/build-site-i18n.ps1
./scripts/validate-i18n.ps1
```

`.github/workflows/ci.yml` 会在 Ubuntu、Windows、macOS 上运行 Rust 门禁，并在 Ubuntu 上运行网站/i18n 门禁。
