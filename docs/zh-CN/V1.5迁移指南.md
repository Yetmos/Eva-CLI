# Eva-CLI V1.5 迁移指南

日期：2026-07-04

V1.5 与 V1.4 兼容。它新增 `eva-release` 和 `eva release ...` 命令组，
但不删除、不重命名 V1.0 到 V1.4 的命令。

## 从 V1.4 升级到 V1.5

1. 重新构建 workspace：

   ```powershell
   cargo build --release
   ```

2. 重新运行 V1.4 生命周期 smoke：

   ```powershell
   cargo run -- upgrade check --output json
   ```

3. 运行 V1.5 聚合发布检查：

   ```powershell
   cargo run -- release check --output json
   ```

4. 查看安全评审：

   ```powershell
   cargo run -- release security --output json
   ```

5. 查看性能预算：

   ```powershell
   cargo run -- release perf --output json
   ```

6. 输出机器可读迁移指南：

   ```powershell
   cargo run -- release migration --from-version 1.4.0 --to-version 1.5.0 --output json
   ```

## 破坏性变更

V1.5 没有破坏性变更。

## 运维说明

- `.eva/tasks` 仍是本地诊断格式，不是持久化公开数据库。
- `restore plan` 仍保持 `apply_allowed:false`。
- `upgrade check` 仍不启动 Supervisor 或 Runtime 进程。
- `hardware bind --apply` 仍只校验逻辑计划。
- V1.5 release checks 不修改项目状态，也不调用外部 provider，适合在 CI 中运行。
