# V1.x 未完整实现功能清单

更新时间：2026-07-08

本文记录 Eva-CLI `V1.7.4-alpha` 中已经具备部分实现、但尚未进入完整真实执行路径的功能。它补充 [当前项目从零到完整实现实施计划](当前项目从零到完整实现实施计划.md)，用于区分“已实现的契约、诊断、计划和受控 envelope”与“仍待补齐的 runtime 能力”。

当前结论：V1.7.4-alpha 已经形成可编译 workspace、稳定 CLI 输出、in-memory basic runtime、外部能力诊断、记忆上下文、硬件绑定计划、备份/生命周期计划、发布加固门禁、第一版 durable backend 布局/迁移锁基线、durable EventBus redrive 基线、durable task snapshot、durable audit sink、filesystem artifact metadata hardening、runtime recovery scanner、event redrive recovery checkpoint、recovery audit smoke、通过 `inspect durable` 暴露的 durable backend diagnostics、受限 Lua VM `on_event` 真实执行边界、只读 Lua context table、Lua host log/audit observation、受控 `ctx.tools.call` capability binding、Lua timeout、instruction budget、cancellation 和 memory budget 执行限制、Lua shadow load/generation route/drain/rollback audit 边界，V1.8 Adapter stdio/http、MCP JSON-RPC/session 和 Skill workflow runner 受控真实执行路径，capability provider selection gate 和 retryable fallback 分类，V1.9.2 policy domain parser/runtime high-risk gate audit 基线，V1.9.3 Discovery 多来源、source report 和增量缓存基线，V1.9.4 durable memory/knowledge、TTL/expiration、compression、redaction 和 index rebuild 基线，V1.9.5 observability file ## 状态口径

| 状态 | 含义 |
| --- | --- |
| 已有边界 | 已有类型、trait、命令、诊断或 plan-first 输出，可以作为后续实现基础。 |
| 部分实现 | 已可在受控条件下运行，但只覆盖 in-memory、mock、simulated、dry-run 或 envelope 路径。 |
| 未完整实现 | 尚未接入真实副作用、持久化、进程管理、外部协议、OS 权限、签名发布或完整策略门禁。 |
| 下一步 | 将现有边界升级为可审计、可回滚、可测试的真实执行路径。 |

## 维护规则

| 规则 | 说明 |
| --- | --- |
| 先测试后扩大副作用 | 从 diagnostic/envelope/plan 升级到真实执行前，先补失败用例和回归测试。 |
| 保持 plan-first | restore、upgrade、hardware、release pointer、权限扩大等高风险路径必须先输出计划和风险。 |
| 不绕过边界 | Lua、Agent、Discovery 不直接拿 provider、设备、文件、socket 或进程句柄。 |
| 文档与实现同步 | 任何真实执行路径落地后，需要同步更新本清单、实施计划、使用手册和 release 文档。 |
