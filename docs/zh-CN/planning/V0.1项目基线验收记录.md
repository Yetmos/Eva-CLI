# V0.1 项目基线验收记录

更新日期：2026-07-02

本文记录 Eva-CLI V0.1 项目基线版本的完成状态。V0.1 的目标不是交付可用 Runtime，而是把后续 V0.2 契约稳定、V0.3 CLI 闭环和 V0.4 最小 Runtime 闭环所需的仓库骨架、文档入口、模块边界和静态配置样例固定下来。

## 1. 版本范围

V0.1 包含：

- 架构文档、官网文档入口、wiki 和中英文文档索引。
- Rust workspace 根配置和 19 个职责 crate。
- 根二进制入口 `src/main.rs` 到 `eva-cli` 的薄转交。
- 静态配置样例，包括 `config/eva.yaml`、Agent manifest、Adapter manifest、Capability manifest、Policy、Routes 和 JSON Schema。
- `cargo test --workspace` 可执行并通过。
- 后续版本实施计划和进度表入口。

V0.1 不包含：

- 真实 CLI 命令解析。
- 最小 Runtime 闭环。
- EventBus、Scheduler、AgentRuntime、Lua host、Adapter、MCP、Memory、Hardware、Backup 或 Lifecycle 的真实运行行为。
- 安装包、release artifact 或跨平台发布验证。

## 4. 验证命令

V0.1 验收使用以下命令：

```powershell
.\scripts\validate-i18n.ps1
cargo test --workspace
```

验收结论：

- i18n 和文档结构校验通过。
- Rust workspace 测试通过。
- 当前 V0.1 只声明项目基线完成，不声明运行时行为完成。

## 5. 移交到 V0.2

V0.2 的第一批任务应从以下内容开始：

1. 为 `eva-policy` 实现权限、sandbox 和 effective policy 的最小类型。
2. 为 routes 和 policy 增加 Rust 类型与 schema 对齐测试。
3. 固定 observability 字段契约。
4. 继续保持 `cargo test --workspace` 作为最低验收门槛。
