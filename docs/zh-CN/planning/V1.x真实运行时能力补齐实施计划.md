# V1.x 真实运行时能力补齐实施计划

更新时间：2026-07-07

本文基于 [V1.x 未完整实现功能清单](V1.x未完整实现功能清单.md)，把 V1.5.1 之后仍停留在 contract、diagnostic、plan-first、in-memory、simulated 或 envelope 路径的能力拆成可执行的小版本计划。目标不是重新定义架构，而是把“未完整实现”逐步升级为可审计、可回滚、可测试的真实执行路径。

当前基线：

- V1.5.1 已完成源码发布与 release hardening checkpoint。
- `eva-cli` 命令面稳定，JSON envelope 和 exit code 是兼容性承诺。
- Runtime 仍以 in-memory basic loop 为主，外部 provider、MCP、Lua VM、硬件、restore/upgrade apply、签名分发还未进入完整真实执行路径。
- 本计划中的版本号是实施里程碑建议；真正发布时仍需遵守 [版本管理方案](../release/版本管理方案.md)。

## 实施约束

| 约束 | 要求 |
| --- | --- |
| 先测试后扩大副作用 | 每次把 plan/envelope/dry-run 升级为真实执行前，先补失败用例、拒绝用例和回归用例。 |
| 保持 plan-first | restore、upgrade、hardware bind、release pointer、权限扩大等高风险路径必须先输出计划和风险。 |
| 不绕过边界 | Lua、Agent、Discovery 不能直接拿 provider、设备、文件、socket 或进程句柄。 |
| 保持兼容 | V1.0-V1.5 已公开命令、JSON 字段、exit code 默认不破坏；新增字段必须可选。 |
| 小步交付 | 每个小版本只包含 3-5 个小功能迭代；每个迭代必须有前置项、实现逻辑和验收口径。 |
| 文档同步 | 任一真实路径落地后，同步本计划、未完整实现清单、使用手册、release notes 和 i18n manifest。 |

## 跨版本依赖图

| 后续能力 | 必须先完成 | 原因 |
| --- | --- | --- |
| 真实 Lua VM | V1.6 durable task/audit | Lua 超时、取消、失败和 generation 切换需要可恢复证据。 |
| 真实 Adapter/MCP/Skill | V1.6 audit、V1.9 policy 基础 | 外部进程和协议调用必须先有审计与权限门禁。 |
| Memory durable context | V1.6 durable backend、V1.7 Lua context | 记忆持久化要先有存储，注入 Lua 前要有安全上下文边界。 |
| 硬件真实 driver | V1.9 policy、V1.9 observability | 真实设备 I/O 涉及 OS 权限和审计，不能只靠 V1.3 plan boundary。 |
| Restore/upgrade apply | V1.6 durable state、V1.9 policy、V1.10 backup signature | 高风险 mutation 必须可锁定、可审计、可回滚。 |
| 签名安装器发布 | V1.10 apply/supervisor | 安装器和升级链路需要真实 runtime apply/handoff 证据。 |

## 统一验收命令

每个小版本完成时至少运行：

```powershell
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo run -- --version
cargo run -- release check --output json
./scripts/validate-i18n.ps1
```

当版本涉及对应能力时追加：

| 能力 | 追加验证 |
| --- | --- |
| Durable runtime | durable store restart smoke；corrupt store recovery test；dead-letter redrive test |
| Lua VM | sandbox escape test；timeout/memory limit test；generation swap/rollback smoke |
| Adapter/MCP/Skill | fake stdio/http provider；fake MCP server；workflow artifact isolation test |
| Policy/Discovery/Memory/Observability | schema invalid fixture；policy denial fixture；slow discovery source；memory redaction；audit export |
| Hardware/Backup/Lifecycle | hardware simulator suite；restore apply dry-run/apply pair；supervisor handoff rollback |
| Release/CLI | installer smoke；signature/provenance check；public JSON contract diff |
