# 网站与文档多语言维护

> Language: 简体中文
> English default entry: [English](../../en/operations/website-docs-i18n.md)
> Content authority: current detailed source

更新时间：2026-07-13

## 当前模型

- 英文是默认公开语言并持有稳定公开 slug。
- 简体中文当前仍是架构和实现规格的详细事实源。
- `/` 与 `/docs/` 默认发布英文；本地化官网页面使用 `/<locale>/`，本地化文档使用 `/docs/<locale>/`。
- 官网 HTML 在构建期生成，发布产物保持为静态 HTML、Markdown 和资源文件。
- 目录名、manifest、HTML `lang` 与 `hreflang` 统一使用 BCP 47 locale code。

## 仓库结构

```text
docs/
  README.md
  _i18n/
    manifest.json
    glossary.json
  assets/
  en/{guide,architecture,capabilities,operations,release,planning,tooling}/
  zh-CN/{guide,architecture,capabilities,operations,release,planning,tooling}/
website/
  _templates/
  _i18n/
  _blog/
  index.html
  zh-CN/index.html
  docs/index.html
```

`docs/_i18n/manifest.json` 是语言、文档 ID、中英文路径、翻译状态和本地化内容图片的机器可读登记表。文件移动时应保持 document ID 稳定。

## 文档生命周期

新增或修改文档时：

1. 架构或实现语义发生变化时，先更新中文详细事实源。
2. 同步英文全文，或明确标注为范围有限的摘要。
3. 在 manifest 中登记两种语言路径和真实翻译状态。
4. 页面属于公开入口时，同步语言文档索引和官网文档卡片。
5. 重新构建并校验生成站点。

删除或合并文档时，应同时移除中英文文件、manifest 和官网条目、所有入站链接，以及只供该文档使用的图片。已被替代的路线图和里程碑历史由 Git 历史、不可变 release tag 和发布说明保存，不继续留在现行文档树中。

## 翻译状态

支持的状态为 `current`、`needs-review`、`stale`、`missing` 和 `partial`。除 `missing` 外，状态必须对应实际存在的翻译路径；缺失翻译不能生成本地化 `hreflang` 页面。

命令、JSON key、Topic 名称、Lua binding、文件路径、错误码和其他机器契约不翻译。共享术语写入 `docs/_i18n/glossary.json`。

## 本地化图片与博客

含可读文字的图片登记在 manifest 的 `assets` 区域。英文图片是默认 source，各语言变体按 locale 映射；Logo 等语言中立资源不需要单独变体。

博客元数据位于 `website/_blog/posts.json`，本地化 HTML 源位于 `website/_blog/content/<locale>/`。构建和校验脚本会检查 post ID、locale、category、slug、内容路径、canonical URL 与 `hreflang`。

## 构建与发布

在仓库根目录运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-site-i18n.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate-i18n.ps1
```

构建脚本从模板、locale JSON、manifest 和博客源重新生成多语言首页、文档索引与博客页面。GitHub Pages 随后发布 `website/`、`docs/` 和共享资源。内容由模板、locale JSON、manifest 或博客源拥有时，不直接手改生成后的 HTML。
