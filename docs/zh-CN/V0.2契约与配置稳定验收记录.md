# V0.2 契约与配置稳定验收记录

更新日期：2026-07-02

本文记录 Eva-CLI V0.2 契约与配置稳定版本的完成状态。V0.2 的目标是把 V0.1 的项目基线推进为后续 Runtime 可以依赖的纯契约层：配置可加载、routes 与 policy 可纳入项目配置、权限策略只能收紧、可观测字段有统一契约。

## 1. 版本范围

V0.2 包含：

- 继续保持 `eva-core` 的零依赖基础契约。
- `eva-config` 加载 `policy` 文档和 `routes` 配置。
- `ProjectConfig` 汇总 agents、adapters、capabilities、policies、routes。
- 项目级校验包含 routes 指向的 Agent。
- `eva-policy` 实现 `PermissionSet`、`SandboxPolicy`、`PolicyLayer`、`EffectivePolicy`。
- `eva-observability` 实现 `TraceFields`、`AuditEvent`、`AuditSink`、`MetricPoint` 等字段契约。
- 对应模块 README 记录 API、边界、测试和后续移交。

V0.2 不包含：

- CLI 命令解析和 `eva validate` 输出。
- Runtime builder、EventBus、Scheduler、AgentRuntime 或 Lua host 行为。
- Adapter、MCP、硬件、记忆、备份或生命周期服务。
- 具体日志后端、数据库 audit sink 或 OpenTelemetry 集成。

## 4. 验证命令

V0.2 验收使用以下命令：

```powershell
cargo fmt --all --check
cargo test -p eva-config
cargo test -p eva-policy
cargo test -p eva-observability
.\scripts\validate-i18n.ps1
cargo test --workspace
```

验收结论：

- V0.2 契约代码可编译、可测试。
- i18n 文档结构校验通过。
- workspace 测试通过。
- 当前版本只声明契约与配置稳定，不声明 CLI 或 Runtime 行为完成。

## 5. 移交到 V0.3

V0.3 应从以下内容开始：

1. 选择并接入 CLI parser。
2. 实现 `eva config validate --output text|json`。
3. 将 `load_project_config`、`EffectivePolicy` 和 `TraceFields` 转换成稳定 CLI diagnostic。
4. 实现 `eva doctor` 的最小环境检查。
5. 实现 `eva inspect` 展示 agents、adapters、capabilities、routes、policy 摘要。
6. 固定 exit code 和 CLI JSON 输出结构。
