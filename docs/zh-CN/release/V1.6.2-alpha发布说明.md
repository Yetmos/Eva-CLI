# Eva-CLI V1.6.2 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.6.2-alpha`

Eva-CLI V1.6.2-alpha 继续推进 V1.6 durable runtime/storage 线。这个版本在
V1.6.1 backend manifest 与 migration lock 布局之上，补齐 filesystem durable
EventLog 和 Durable EventBus redrive 基线。

## 重点

- 新增 `FileSystemEventLog`，记录保存在 durable backend 的 `events/log` 目录。
- 持久化 publish、ack、fail、consumer、结构化错误、target、payload 和 event
  metadata 字段，重新打开 backend 后仍可按 sequence 查询。
- 新增 `DurableEventBus` 与 `FileSystemDeadLetterStore`，把可查询 dead-letter
  记录保存到 `events/dead_letters`。
- 新增 durable redrive，生成 `:replay-N` 子事件 ID，并重新 publish 为新的投递尝试。
- 新增默认 `RedrivePolicy` 字段：`retry_delay_ms` 与 `next_attempt_after_ms`。
  字段已经进入序列化格式；真实延迟调度仍留给后续 runtime/scheduler 节点。
- 在 V1.x 真实运行时能力补齐实施计划中把 V1.6.2 标记为完成。

## 兼容性

V1.6.2-alpha 不改变公开 CLI 命令名、JSON envelope 形状、exit code 或既有
V1.5/V1.6.1 发布命令。它仍是 alpha 检查点，因为 durable task/audit store、
runtime crash recovery、scheduler backoff dispatch 和 durable smoke 诊断仍在后续节点中。
