# Eva-CLI V1.11.5 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.11.5-alpha`

Eva-CLI V1.11.5-alpha 完成 CLI runtime command completion 切片。它补齐 typed
event 发布、Agent lifecycle evidence 和 capability provider routing 的 operator
命令面，同时保持稳定 JSON envelope、trace 字段、exit code 映射和 plan-first
安全边界。

## 重点

- 新增 `eva emit`，可构造 typed `eva-core::Event`，并发布到 in-memory EventBus
  边界或 V1.6 durable backend EventBus log。
- 新增 `eva agent status`、`eva agent drain` 和 `eva agent reload`，复用
  ProjectConfig manifest、`AgentRuntime`、`AgentLifecycle`、`DrainCoordinator`
  和 `GenerationController` 输出 lifecycle、drain 和 generation reload evidence。
- 新增 `eva capability list`、`eva capability probe` 和 `eva capability call`，
  复用 `CapabilityRegistry`、provider selection plan、permission gate、runtime
  policy decision、`CapabilityRouter` 和 `AdapterBackedCapabilityHost`。
- `capability call` 默认 dry-run。只有 `--confirm <request-id>` 与 request id
  匹配时才执行；provider 超出 capability manifest allowlist 会在调用前拒绝。
- 将 release metadata、release readiness docs、i18n manifest 和静态官网内容同步到
  `1.11.5-alpha` 检查点。

## 兼容性

V1.11.5-alpha 对公开 CLI 命令、JSON envelope、exit code 和既有 release evidence
gate 保持增量兼容。既有命令组输出形状不变。新增 Agent 命令只是 lifecycle evidence
边界，不会重启 daemon、修改 live scheduler 或执行真实 hot reload apply。确认后的
capability call 仍是受控 CLI invoke，并继续输出 `mutation_executed:false`。

它仍是 alpha 检查点。retention/rotation、db sink policy、OS provider
supervision、真实硬件接入、生产 daemon 驱动的 hot-reload orchestration、签名安装器、OS
包仓库和生产 signing/attestation credential 仍属于后续 release 范围。V1.15.8 补齐
policy-driven memory redaction/audit JSONL evidence；V1.16.1 补现有 JSONL best-effort
runtime audit wiring；V1.16.2 补 tracing subscriber bridge 到现有 JSONL/dev-console sink；
V1.16.3 补显式 OpenTelemetry SDK OTLP trace/metrics exporter smoke 和 collector degraded report。
这些仍不等于带 retention/db sink 策略的完整生产观测后端或生产检索调度。
