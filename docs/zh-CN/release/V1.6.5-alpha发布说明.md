# Eva-CLI V1.6.5 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.6.5-alpha`

Eva-CLI V1.6.5-alpha 完成 V1.6.5 durable diagnostics and smoke gate 切片。
这个版本建立在 V1.6.1 到 V1.6.4 的 durable backend、EventBus、task/audit、
artifact 和 recovery evidence 之上，补齐稳定的只读诊断报告、`inspect.durable`
CLI envelope，以及 CI/release workflow 中的 durable backend inspection smoke。

## 重点

- `FileSystemEventLog`、`FileSystemDeadLetterStore` 和 `DurableEventBus` 新增
  read-only 打开路径，诊断读取空 durable store 时不会创建 `events/log` 或
  `events/dead_letters` 目录。
- 新增 `eva_runtime::inspect_durable_backend()` 和 `DurableDiagnosticsReport`。
- 诊断报告输出 durable backend path、mode、schema version、layout version、
  migration status、event log count、dead-letter count 和 `pending_redrive_count`。
- `pending_redrive_count` 只统计已到期且原始 event 未 ack 的 redrive candidate。
- 新增 `eva inspect durable --durable-backend <path>`，同时提供 text 输出和稳定
  `inspect.durable` JSON envelope。
- `release check` 新增 `REL-DURABLE-DIAGNOSTICS-001` 门禁；CI/release workflow
  会创建 durable backend 并执行 `inspect durable` smoke。
- release workflow 会捕获 `inspect-durable.json` 作为发布证据。

## 兼容性

V1.6.5-alpha 是增量版本。现有公开 CLI 命令、JSON envelope 形状和 exit code
保持兼容。新增 `inspect durable` 命令要求显式传入 `--durable-backend <path>`，
并以 read-only 模式打开 backend。

它仍是 alpha 检查点，因为完整 runtime audit wiring、scheduler backoff dispatch、
provider process recovery、真实 apply gate、签名安装器、系统包和 provenance bundle
仍属于后续工作。
