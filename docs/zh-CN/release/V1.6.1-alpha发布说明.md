## 此 tag 包含

- `FileSystemDurableBackend`、`backend.manifest`、schema version `1`、
  `eva.durable.v1` 布局和稳定的 store 目录。
- 读写 `migration.lock` 处理，包括校验失败后的清理。
- 不创建文件、不获取 migration lock 的只读校验路径。
- `InMemoryDurableBackend` 继续用于测试和兼容。

## 未包含

此 tag 不包含 durable EventBus、task/audit store、recovery coordinator 或 durable
diagnostics；这些切片由后续 V1.6 tag 提供。它也不包含破坏性 apply 或签名分发。

## 复现发布

```powershell
git switch --detach v1.6.1-alpha
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo test -p eva-storage
./scripts/validate-version-management.ps1 -Tag v1.6.1-alpha
cargo run -- --version
cargo run -- release check --output json
```

## 制品

- GitHub Release 提供自动生成的源码归档，上传的 asset 为 **0**。
- Actions 保留 release/package evidence，以及 Linux、Windows、macOS 的未签名
  native archive 和对应 evidence；这些 artifact 独立于 Release 到期。
- GHCR 包含 `ghcr.io/yetmos/eva-cli:1.6.1-alpha`。

`release check` 中的 `REL-DURABLE-BACKEND-001` 是编译进代码的 readiness
声明，不会执行远端 workflow 或生产 migration。
