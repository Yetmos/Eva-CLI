# Eva-CLI V1.5 发布加固

日期：2026-07-04

V1.5 把 1.x 最后一段发布准备工作落成可执行检查，而不是只保留人工
checklist。当前仍是源码发布 checkpoint，但已经具备跨平台、安全评审、性能预算、
迁移步骤、兼容策略和发布说明的公开命令面。

## 支持命令

```powershell
cargo run -- release check --output json
cargo run -- release check --target windows --output json
cargo run -- release security --output json
cargo run -- release perf --output json
cargo run -- release migration --output json
```

## 发布门禁

| 门禁组 | 范围 | 阻断规则 |
| --- | --- | --- |
| 跨平台 | Windows、Linux、macOS CI smoke 命令、shell 和路径假设 | 任一必需平台门禁 blocked 都阻断发布就绪。 |
| 稳定性 | task status/logs/cancel、timeout dead-letter replay、restore plan、upgrade rollback plan | 恢复路径必须可审计。 |
| 文档 | V1.5 发布文档、迁移指南、兼容策略、模块 README | 公开文档必须和可执行命令面一致。 |
| 安全 | policy、Lua sandbox、secret redaction、MCP allowlist、hardware handle、lifecycle apply 风险 | high/critical blocked finding 会阻断发布就绪。 |
| 性能 | EventBus、Scheduler、Adapter probe、memory context、backup、release check 预算 | 任一必需性能预算超限都阻断发布就绪。 |
| 迁移 | V1.4 -> V1.5 迁移指南和兼容策略 | 破坏性变更必须有文档和测试。 |

## 安全评审

`release security` 覆盖：

- effective policy 会收窄权限并拒绝请求扩权；
- Lua host 接收受控上下文快照，而不是宿主句柄；
- backup manifest 保留敏感条目的 redaction 元数据；
- MCP probe 只走 allowlist，不启动外部 server；
- hardware discovery 不从 CLI 授予 raw I/O handle；
- restore 和 upgrade apply path 在签名 artifact 与显式 apply 授权之前保持 warning tracked。

## 非目标

V1.5 不提供安装包、签名 release artifact、真实 MCP 进程执行、真实硬件 I/O、
destructive restore 或真实 Supervisor 进程管理。这些能力仍是后续版本工作，不能从
`release` 命令组中推断为已支持。
