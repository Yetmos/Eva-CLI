# Eva-CLI V1.5 兼容性策略

日期：2026-07-04

V1.5 明确源码发布面的兼容承诺：保护已经依赖 V1.0-V1.4 命令面的脚本和 CI，
同时增加发布加固命令。

## 稳定契约

| 契约 | V1.5 策略 |
| --- | --- |
| JSON envelope | 成功输出继续保留 `ok`、`command`、`exit_code`、`data`、`trace`；错误输出继续保留 `ok`、`command`、`exit_code`、`error`、`trace`。 |
| Exit code | `0`、`1`、`2`、`3`、`4`、`5`、`64` 保持既有含义。 |
| 命令面 | V1.5 只新增 `release check/security/perf/migration`；旧命令继续可用。 |
| 配置样例 | 现有 sample manifest 不需要迁移即可加载。 |
| Plan-first 操作 | hardware bind、restore、upgrade 继续保持非破坏性，除非未来版本明确新增 apply gate。 |
| 废弃窗口 | 破坏性 CLI 或 manifest 变更需要至少一个已记录发布窗口。 |

## 公开命令族

- `eva --version`
- `eva version --output json`
- `eva doctor`、`eva config validate`、`eva inspect`
- `eva run --example basic`
- `eva task status/logs/cancel`
- `eva adapter`、`eva mcp`、`eva skill`、`eva discovery`
- `eva memory context`
- `eva hardware list/probe/bind`
- `eva backup create`、`eva snapshot create`、`eva restore plan`、`eva upgrade check`
- `eva release check/security/perf/migration`

## 兼容边界

本策略覆盖源码 checkout、CLI 命令、JSON envelope 和文档化 sample 配置；不承诺
安装包格式、签名 artifact 布局、生产性能数字或持久化存储 schema 稳定。
