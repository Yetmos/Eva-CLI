# Eva-CLI V1.7.2 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.7.2-alpha`

Eva-CLI V1.7.2-alpha 完成 V1.7.2 Lua host binding 切片。它基于 V1.7.1
真实 Lua VM 边界，补齐只读 request/trace/memory context table、Lua host
observability，以及通过 `CapabilityHostApi` 受控执行的 `ctx.tools.call` 路径。

## 重点

- 新增只读 `ctx.request`、`ctx.trace` 和 `ctx.memory` 表，同时保留 V1.7.1
  顶层 context count 字段以兼容旧脚本。
- 移除 Lua 全局 `rawset` 入口，避免脚本绕过只读 context snapshot。
- 新增 `ctx.host.log(level, message)` 和 `ctx.host.audit(message)`，生成带
  trace 的 `LuaHostObservation`，由 runtime task log 和 CLI JSON 输出。
- 新增 `LuaHost::run_on_event_with_tools` 和 `ctx.tools.call(capability,
  value)`，通过 `CapabilityHostApi` 调用 capability，不向 Lua 暴露 raw
  provider、file、socket、process、memory service、knowledge service 或 audit
  sink handle。
- Lua nil、boolean、number、string、array 和 object table 会转换为
  JSON-compatible text payload 后进入 capability invocation。
- 工具调用返回受限 Lua table，仅包含 `request_id`、`status`、`ok`、`output`、
  `error` 和 `error_kind`。
- unknown 或 disabled capability 会在 Lua host boundary 内被拒绝。
- basic 示例 Lua 脚本改为直接通过 `ctx.tools.call` 调用 `config.lint`，不再
  依赖返回 legacy `capability` 字段后由 runtime 二次调用。
- `release check` 新增 `REL-LUA-HOST-BINDINGS-001` 发布门禁。

## 兼容性

V1.7.2-alpha 对公开 CLI 命令、JSON envelope 和 exit code 保持增量兼容。
Legacy `LuaEventResult` 字段、V1.7.1 静态 parser compatibility fallback、durable
diagnostics 和 release checks 继续兼容。

它仍是 alpha 检查点，因为 Lua timeout/instruction budget、memory limit、shadow
load、generation swap、rollback、真实 provider execution、签名安装器、系统包、
destructive apply 路径和 provenance bundle 仍属于后续工作。
