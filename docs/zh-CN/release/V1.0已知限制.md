# Eva-CLI V1.0 已知限制

V1.0 是 core CLI 发布，不代表所有架构文档都已实现。它稳定已有可执行路径、补齐文档和 CI 门禁，并明确高级能力的边界。

## V1.0 Core 支持范围

- `eva --version` 和 `eva version --output json`。
- `eva doctor`、`eva config validate`、`eva inspect`。
- 通过 `in_memory_v1.0` 执行 `eva run --example basic`。
- 通过 `eva task status`、`eva task logs`、`eva task cancel` 读取本地 task 诊断。
- 结构化 JSON envelope、trace 字段、结构化错误和稳定 exit code。
- basic 示例中的 in-memory EventBus、Scheduler、AgentRuntime、受控 Lua host contract、builtin `config.lint` capability、dead-letter replay 诊断和 task logs。

## V1.0 Core 不支持范围

- 后台 daemon 或长生命周期任务调度器。
- durable task database、崩溃恢复、durable redrive queue 或 durable replay cursor。
- 真实 Lua VM、任意 Lua 求值、真实 hot-reload VM swap 或多 generation drain。
- 外部 Adapter 执行、stdio/http transport、MCP client/server、Skill 执行或硬件 Adapter 调用。
- Discovery scan、memory service、knowledge service、backup、restore、migration package、release snapshot、supervisor lifecycle 或进程级升级。
- 安装包、签名二进制、release channel 或自动更新。
- `.eva/tasks` 兼容性保证；它只是本地诊断格式，不是稳定数据库 API。

## 运行说明

- V1.0 命令是同步单进程命令。
- basic 示例是刻意收窄的回归基线。
- V1.1+ 架构文档仍是设计目标，不应被解读为 V1.0 runtime 承诺。
- CI 证明源码 checkout 路径，不证明打包分发 artifact。
