## 范围

V1.0 是内部 core CLI 能力里程碑。它建立了范围较窄的同步源码 checkout 路径和第一批稳定 CLI 输出契约。仓库中没有 `v1.0` Git tag，因此本文档不能作为 V1.0 已单独公开发布的证据。

当前实现缺口统一维护在 [V1.x 未完整实现功能清单](../planning/V1.x未完整实现功能清单.md)，而不在这份历史记录中追加。

## V1.0 里程碑已实现范围

| 领域 | 历史边界 |
| --- | --- |
| CLI 基础 | `eva --version`、`eva version --output json`、`doctor`、`config validate` 和 `inspect` |
| 基础执行 | 通过 `in_memory_v1.0` composition 执行 `eva run --example basic` |
| Task 诊断 | `.eva/tasks` 下的本地 `task status`、`task logs` 和 `task cancel` 记录 |
| 公开输出 | 结构化成功/错误 envelope、trace 字段和 exit code 映射 |
| Runtime 基线 | 内存 EventBus、Scheduler、AgentRuntime、受控 Lua host contract、`config.lint`、dead-letter 诊断和 basic task logs |

这些命令都是同步单进程路径。basic 示例是回归基线，不是生产 runtime。

## 该里程碑之外

在提交 `437087c` 中，以下能力尚未端到端实现：

- durable task/event 存储和崩溃恢复；
- 真实 Lua VM 执行和热重载；
- 外部 Adapter、MCP、Skill 和硬件执行；
- discovery、memory、knowledge、backup、restore、snapshot 和 upgrade 工作流；
- 后台服务管理和打包分发。

上述多个领域在当前 `main` 中已有受控 alpha 实现，其实际边界分别可能是可调用行为、文件系统持久化、合成 fixture、仅 library/test 契约或尚缺生产集成。因此这份历史清单不能被复用为当前 backlog。

## 证据边界

- V1.0 里程碑只证明源码 checkout 和本地回归路径。
- `.eva/tasks` 是本地诊断格式，不是持久化公开数据库契约。
- 当时的架构文档描述目标，不是该里程碑已交付的保证。
- 安装包、签名、平台服务、硬件和外部 provider 证据不在本快照中。
