# Eva-CLI V1.7.4 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.7.4-alpha`

Eva-CLI V1.7.4-alpha 完成 V1.7.4 Lua hot-reload lifecycle 切片。它基于
V1.7.1 真实 Lua VM 边界、V1.7.2 host binding 和 V1.7.3 resource limit，
补齐 shadow-load 健康检查、generation route gate、旧 generation drain
evidence、rollback audit record，以及 hot-reload 生命周期发布门禁。

## 重点

- 新增 `LuaShadowLoader`、`LuaShadowCandidate`、`LuaShadowLoadReport` 和脚本级
  shadow report，用于 generation promote 前的 dry-run health check。
- shadow-load 脚本默认带 timeout、instruction budget、memory budget 和 no-op
  capability host 边界，健康检查不会切换 scheduler route，也不会调用真实
  provider 路径。
- 新增 `GenerationRouteGate`，candidate generation 通过 shadow health 之前，新
  work 仍写入 active generation。
- 新增 `GenerationDrainEvidence` 和 generation swap drain 计划；切换后旧
  generation 会记录 `accepts_new_work=false`、in-flight 数量、完成状态和 audit
  evidence。
- 新增 `plan_generation_lifecycle_rollback()`，候选 generation 失败时保持
  scheduler route 指向 previous healthy generation，并把 drain evidence 纳入
  rollback audit。
- `release check` 新增 `REL-LUA-HOT-RELOAD-001` 发布门禁和
  `lua_hot_reload_lifecycle_ready` audit marker。

## 兼容性

V1.7.4-alpha 对公开 CLI 命令、JSON envelope、exit code、Lua host binding 和
resource-limit 行为保持增量兼容。既有脚本继续通过受限 `on_event(ctx)` 执行边界运行。

它仍是 alpha 检查点，因为真实 provider execution、完整 runtime audit wiring、常驻
daemon 驱动的 hot-reload orchestration、签名安装器、系统包、destructive apply 路径
和 provenance bundle 仍属于后续 release 范围。
