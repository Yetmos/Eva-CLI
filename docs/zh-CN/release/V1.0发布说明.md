# Eva-CLI V1.0.0 发布说明

日期：2026-07-03

Eva-CLI V1.0.0 是第一个 core release checkpoint。它把当前可执行 CLI、配置校验、in-memory basic runtime loop、task diagnostics、文档和 CI 门禁整理为可由新用户从源码安装并跑通的发布面。

## 新增

- `eva --version` 和 `eva version --output json` 报告 V1.0 core 发布面。
- `RuntimeBuilder::in_memory_v10()` 和 `RuntimeMode::InMemoryV10` 将 V1.0 basic runtime 标识为 `in_memory_v1.0`，generation 为 `basic-v1.0`。
- GitHub Actions CI 覆盖格式检查、clippy、workspace 测试、V1.0 quickstart smoke commands 和网站/i18n 校验。
- 在 `docs/en/` 与 `docs/zh-CN/` 中补齐 V1.0 quickstart、known limitations 和 release notes。

## 稳定

- `doctor`、`config validate`、`inspect`、`run --example basic` 和 `task status/logs/cancel` 是 V1.0 core CLI contract。
- JSON success/error envelope、trace 字段、结构化错误和 exit code 映射保持为公开诊断形态。
- `examples/basic/` 是 V1.0 支持的端到端示例和回归基线。

## 明确非目标

- V1.0 不执行外部 Adapter/MCP/Skill/Hardware。
- V1.0 不实现真实 Lua VM、durable task database、daemon、supervisor、backup/restore 或 release snapshot。
- 本 checkpoint 不提供安装包或签名 release artifact。

## 验证

V1.0 release gate：

```powershell
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo run -- --version
cargo run -- version --output json
cargo run -- doctor --output json
cargo run -- config validate --output json
cargo run -- inspect runtime --output json
cargo run -- run --example basic --task-id req-release-v10 --output json
cargo run -- task status --task req-release-v10 --output json
cargo run -- task logs --task req-release-v10 --output json
cargo run -- run --example basic --task-id req-release-timeout --timeout-ms 0 --replay-dead-letters --output json
cargo run -- run --example basic --task-id req-release-cancel --cancel --output json
./scripts/build-site-i18n.ps1
./scripts/validate-i18n.ps1
```
