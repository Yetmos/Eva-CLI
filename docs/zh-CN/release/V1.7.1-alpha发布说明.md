# Eva-CLI V1.7.1 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.7.1-alpha`

Eva-CLI V1.7.1-alpha 完成 V1.7.1 Lua VM execution boundary 切片。
这个版本把 `eva-lua-host` 从静态 `on_event` 字段解析升级为真实 `mlua` VM adapter，
同时保持既有 `LuaEventResult`、sandbox 和兼容性契约。

## 重点

- `eva-lua-host` 新增 `LuaVmAdapter` 和 `MluaVmAdapter`。
- 通过 `mlua` 使用 vendored Lua 5.4，并设置 `default-features = false`。
- 第一版 VM 边界只加载 table、string、utf8 和 math 标准库；不加载 `os`、`io`、
  `package` 和 `debug`。
- 支持执行 `return root`、global `on_event` 或 `root.on_event` 形态的真实 Lua
  handler。
- 向脚本注入受控 event table 和 V1.2 context 计数快照，不暴露 raw file、socket、
  process、memory service 或 provider handle。
- Lua 返回 table 继续转换为既有 `LuaEventResult` 字段，保持 basic runtime 和 CLI
  JSON 输出兼容。
- 编译错误映射为 `lua_syntax_error`，handler 失败映射为 `lua_runtime_error`，错误消息不泄漏宿主文件系统路径。
- 对不符合 Lua chunk、但匹配旧受控 contract 的 legacy 静态字段脚本保留
  compatibility fallback。
- `release check` 新增 `REL-LUA-VM-EXECUTION-001` 发布门禁。

## 兼容性

V1.7.1-alpha 对公开 CLI 命令、JSON envelope 和 exit code 保持增量兼容。
现有 `run --example basic`、task diagnostics、durable diagnostics 和 release checks
继续兼容。

它仍是 alpha 检查点，因为 `ctx.tools`、`ctx.host`、timeout/instruction budget、
memory limit、shadow load、generation swap、rollback、真实 provider execution、签名安装器、系统包、destructive apply 路径和 provenance bundle 仍属于后续工作。
