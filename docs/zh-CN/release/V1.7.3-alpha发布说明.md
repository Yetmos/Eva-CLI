# Eva-CLI V1.7.3 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.7.3-alpha`

Eva-CLI V1.7.3-alpha 完成 V1.7.3 Lua resource limit 切片。它基于 V1.7.1
真实 Lua VM 边界和 V1.7.2 host binding，把 Lua 执行接入 wall-clock timeout、
instruction budget、cancellation token 和 memory budget 四类可中断边界。

## 重点

- 新增 `LuaExecutionLimits`，统一配置 wall-clock timeout、instruction budget、
  cancellation token 和 memory limit。
- 通过 `mlua` hook 中断长时间运行的 Lua 脚本，避免脚本长期占用 runtime。
- Lua 超时映射为稳定 `ErrorKind::Timeout` / `lua_timeout` evidence。
- 指令预算耗尽映射为稳定 `lua_instruction_budget_exceeded`，错误上下文记录
  配置的 budget。
- 新增 `LuaCancellationToken`，并通过 basic runtime options 传递取消状态，确保
  脚本取消后不会继续执行后续 capability call。
- 当存在 memory budget 时调用 `Lua::set_memory_limit`，Lua 内存分配失败会映射为
  `lua_memory_limit_exceeded`。
- `release check` 新增 `REL-LUA-RESOURCE-LIMITS-001` 发布门禁。

## 兼容性

V1.7.3-alpha 对公开 CLI 命令、JSON envelope、exit code 和 Lua host binding 保持增量兼容。
未显式配置限制的既有脚本继续通过默认兼容路径执行。

它仍是 alpha 检查点，因为 shadow load、generation swap、rollback、真实 provider
execution、签名安装器、系统包、destructive apply 路径和 provenance bundle 仍属于后续工作。
