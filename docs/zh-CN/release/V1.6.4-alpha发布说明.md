# Eva-CLI V1.6.4 Alpha 发布说明

状态：alpha 预发布
Tag：`v1.6.4-alpha`

Eva-CLI V1.6.4-alpha 完成 V1.6.4 runtime crash recovery coordinator
切片。这个版本建立在 V1.6.1 到 V1.6.3 的 durable backend、filesystem
EventLog、durable dead-letter redrive、task snapshot、audit sink 和 artifact
evidence 之上，补齐重启恢复决策、受控 event redrive checkpoint 和 durable
recovery audit smoke。

## 重点

- 新增 `FileSystemTaskStateStore::list_snapshots()` 和
  `RuntimeRecoveryCoordinator`，恢复流程可以扫描 durable task snapshot，并把
  未完成的 `queued` / `running` task 标记为 `interrupted` 或 `recovering`。
- `DurableEventBus::redrive_dead_letter` 支持单事件 durable redrive，并保留
  `RedrivePolicy` 证据。
- recovery checkpoint 只 redrive 同时具备 task dead-letter evidence、durable
  dead-letter record、未 `acked` event log record、且 `next_attempt_after_ms`
  已到期的事件。
- 已 ack、未到期、缺失证据和非法 event id 的 redrive candidate 会进入明确的
  skipped evidence。
- redrive 成功后把 receipt 写回 task snapshot 的 `replayed_events`。
- 新增 `AuditAction::RuntimeRecovered`，并通过 `FileSystemAuditSink` 写入
  durable recovery audit record。
- `release check` 新增 `REL-DURABLE-RECOVERY-001` 门禁。

## 兼容性

V1.6.4-alpha 不改变公开 CLI 命令名、JSON envelope 形状或 exit code。新增
recovery API 和 release gate 都是增量能力。`release check` 会额外输出 durable
recovery evidence，migration 仍保持从 V1.5.1 兼容。

它仍是 alpha 检查点，因为真实 provider 进程恢复、scheduler backoff 调度、完整
runtime 状态恢复、SQLite/WAL 索引、签名安装器、provenance bundle 和 destructive
apply gate 仍属于后续工作。
