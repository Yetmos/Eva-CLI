## 此 tag 包含

- `RuntimeRecoveryCoordinator` 扫描 durable task snapshot，并把未完成任务分类为
  interrupted 或 recovering。
- 受控单事件 redrive 要求 task evidence、dead-letter record、未 ack event 和
  已到期 retry checkpoint 同时存在。
- 成功的 redrive receipt 写入 `replayed_events`。
- `AuditAction::RuntimeRecovered` 和编译进代码的
  `REL-DURABLE-RECOVERY-001`。

## 未包含

这是 recovery decision/evidence 基线，不是完整 process/provider recovery。
它不包含 scheduler dispatch、完整 runtime 状态恢复、SQLite/WAL 索引、破坏性
apply 或生产 recovery 验证。

## 制品

- GitHub Release 提供自动生成的源码归档，上传的 asset 为 **0**。
- Actions 保留 release/package evidence，以及 Linux/Windows/macOS 未签名

`release check` 评估内置 recovery evidence，不会执行真实 crash-recovery 部署。
