# Backup, Migration, and Release Snapshot

Backups, migration packages, and release snapshots are Runtime safety
capabilities, not Agent-owned features.

Canonical source:
[docs/en/backup-migration-release-snapshot.md](https://github.com/Yetmos/Eva-CLI/blob/main/docs/en/backup-migration-release-snapshot.md)

## Decision

Eva-CLI should place trusted execution for these operations inside Runtime
services. Agents may request operations and explain outcomes, but they must not
own the actual file mutation, artifact verification, restore, rollback, or
release pointer movement.

```text
Agent
  -> interpret intent
  -> request Runtime operation
  -> explain result

Runtime
  -> enforce policy
  -> lock and stage changes
  -> create manifests and checksums
  -> verify artifacts
  -> commit, restore, or rollback
  -> write audit records
```

## Runtime Responsibilities

Runtime owns:

- backup scope resolution and workspace boundary checks
- migration package schema and compatibility validation
- release snapshot capture and comparison
- manifest, checksum, and optional signature verification
- dry-run versus apply separation
- operation locks, leases, and recovery states
- restore and rollback authority
- retention, quarantine, and artifact cleanup
- structured audit records and trace IDs

## Agent Responsibilities

Agents may:

- recommend when a backup or snapshot is useful
- draft migration descriptions and release notes
- call controlled Runtime APIs
- explain preflight warnings and policy denials
- summarize artifact IDs, verification evidence, and audit IDs
- analyze Runtime failure logs

Agents should not bypass Runtime policy or directly edit files to restore,
apply, or roll back state.

## Artifact Contract

Every backup, migration package, and release snapshot should have a manifest
that records artifact ID, type, actor, request ID, runtime generation, resolved
scope, schema version, compatibility data, entries, checksums, policy decision,
verification result, and audit ID.

The manifest is generated by Runtime code. Agent-provided metadata can enrich
the record, but it must not be trusted as final scope or verification evidence.

## Relationship to Releases

Release snapshots connect the active release pointer, Runtime generation,
configuration digest, policy digest, manifest registry digest, Lua capability
generation, AdapterRegistry generation, State Store schema, EventBus watermark,
health checks, associated migration packages, backup artifacts, and rollback
eligibility.

Rollback can be Agent-assisted, but capture, comparison, pointer movement, and
rollback execution remain Runtime-controlled.
