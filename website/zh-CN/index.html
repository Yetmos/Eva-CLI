开源 Agent 运行时

Eva-CLI

面向多 Agent 自动化的受控运行时

Eva-CLI 用 Rust 承载可靠运行时边界，用 Lua 承载可热更新 Agent 行为，并通过统一 Topic EventBus 连接外部 Agent、MCP、Skill、记忆、知识库与硬件能力。

Rust 运行时边界 Lua 热更新 Topic EventBus
阅读文档 总体架构 开发进度 讨论 反馈
执行模型 受控自动化链路
  1. Rust Core 托管 policy、记忆、恢复和进程边界。
  2. Lua Agents 热更新 Agent 行为，同时保持能力访问显式可控。
  3. Adapters 通过权限边界接入 CLI、MCP、HTTP、Skill 与硬件。

架构文档围绕实现规划和评审组织。

当前阶段

架构和方案文档

架构方案和模块划分方案已经补齐。下一步实际工作是创建 Rust workspace、固化可执行契约、搭建最小 CLI 骨架，并验证一条端到端 Runtime 闭环。

  1. 已完成
    架构和模块划分

    系统边界、运行时模型、Adapter、记忆、发现、硬件、恢复、官网、文档结构和模块划分方案已经建立。

  2. 当前
    Workspace 骨架和契约定义

    下一步是创建 Rust workspace、manifest schema、event 类型、policy 规则、错误模型和 Lua host API 契约。

  3. 下一步
    最小可运行骨架

    创建 CLI 入口、配置读取、Runtime 初始化、no-op 实现，以及 CI 构建和测试闭环。

  4. 后续
    Runtime 闭环和模块实现

    验证 Ingress -> EventBus -> Scheduler -> Lua Agent -> Rust Tool，再按模块逐个补齐。

下一步实现工作
  • 按模块划分方案创建 Rust workspace。
  • 定义 manifest、event、policy、error 和 Lua host API 契约。
  • 搭建最小可运行 CLI 骨架和 Runtime 闭环。
